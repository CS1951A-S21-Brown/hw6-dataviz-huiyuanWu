List 3 reasons why D3 was helpful and improved your visualization (6 points)

1. D3 provides convenient ways reads data in and cooperate with Javascript's anonymous function.
2. D3 cooperates with html and css, so I can easily customize my visualization.
3. D3 provides a variety of external libraries to solve specific problems, like country maps.

List 3 reasons why D3 would not be the best tool for creating a visualization (6 points)

1. Comparing to matplotlib or matlab, D3 lacks the ability to deal with big data and provided scientific computing functions.
2. D3 requires background knowledge of javascript and web DOM, which might not be in the technique stack of a data scientist.
3. D3 is still evolving fast, and there's no tutorial from the official website about the latest version.

Accessible Data Visualization (10 points)

Users with full sights will find those dashboards accessible, while users with visual disabilities might have difficulties to access the dashboards. To make the data chart accessible to blind people, I will add screen reader to the website, just as mentioned in “Why Accessibility Is at the Heart of Data Visualization”. And also, add more accessible design in my html to let user navigate through tab. A screen reader should help the visual disable users to better understand what the dashboard is about. Also, include alt text in the HTML will also help those users, since sometimes a dashboard is too complex to access by a screen reader. When that's the case, alt text can briefly describe what's that dashboard is about.

I could add tabIndent to the html divs, and also include alt text for those svg. But I didn't implement that. One reason for not making my dashboard more accessible is I have to put a lot of extra works to send the same message by text. It is not easy to summarize the dashboards, especially the third one, which is basically every match in the latest 2 world cups.
